# Knowledge Graph Creator — 知识图谱生成器

## 概述

扫描项目中的可读文本文档（源代码、配置文件、文档等），自动分析文件之间的关联关系，**提取函数/类定义及注释内容**填充节点描述，输出交互式 D3.js 力导向图 HTML 文件。

用户只需说"帮我创建项目知识图谱"，即可自动完成从扫描到可视化的全流程。

### 支持两种项目类型

| 项目类型 | 判断条件 | 处理策略 |
|---------|---------|---------|
| **代码项目** | 代码文件 > 40% | 分析 import/extends + 提取函数注释/docstring |
| **文档项目** | .md/.txt > 60% | 按目录层级 + Markdown 内部链接构建 |
| **配置项目** | .yaml/.json > 60% | 分析配置间引用（include/extends/depends_on） |
| **混合项目** | 无明显主导 | 综合使用以上策略 |

## 触发条件

用户说以下任意话时自动调用：

## 工作流程

### Step 1：扫描项目文件

扫描项目根目录下所有可读文件（支持 30+ 种格式），自动跳过隐藏目录（以 `.` 开头）、node_modules、vendor、dist、build、__pycache__、target 等无关目录。

### Step 2：判断项目类型

根据文件类型分布自动判断是代码项目、文档项目、配置项目还是混合项目，决定分析策略。

### Step 3：分析关联关系

| 关系类型 | 检测方式 | 示例 |
|---------|---------|------|
| **导入/引用** | import / require / include | `import { Component } from './Component'` |
| **继承** | extends / implements | `class Service extends BaseService` |
| **函数/类定义** | 提取函数签名、类名、方法名 | `def process_data():` / `class UserService:` |
| **文档注释** | 提取 docstring 和关键注释 | `"""处理用户登录逻辑"""` |
| **Markdown 链接** | 文档之间的内部链接 | `[详见架构说明](./architecture.md)` |
| **配置引用** | 配置文件依赖关系 | `config.yaml include: base.yaml` |
| **目录归属** | 目录结构反映的模块关系 | `controllers/` / `docs/guide/` |
| **数据流** | Controller → Service → Entity | 根据 import 调用链推断 |

**效率策略**：
- 小项目（< 50 个文件）：逐文件读取，完整分析
- 大项目（≥ 50 个文件）：只读取 import 行和函数签名行

### Step 4：提取注释内容填充节点描述

这是本 Skill 的核心能力之一。从文件中提取有意义的注释/文档内容，填入节点的 `description` 字段：

**提取优先级：**
1. 文件级 docstring / 模块注释
2. 类级 docstring + 类名
3. 函数/方法签名 + 行内注释
4. 无注释时：文件名语义推断，或用路径最后两级推测功能

**示例：**
```python
# Python：提取模块 docstring
"""用户权限管理模块
提供用户登录、鉴权、角色管理功能
"""
class AuthService:
    """认证服务，处理 JWT 令牌和会话管理"""
```

```javascript
// JS/TS：提取 JSDoc 注释
/**
 * 用户认证中间件
 * 验证 JWT token 并注入用户信息到 request
 */
```

```markdown
# 文档项目：提取标题作为节点描述
## API 接口说明
此处记录了所有 REST API 的接口定义和参数说明
```

### Step 5：非代码项目处理

对于纯文档、Wiki、配置仓库：

- **Markdown 链接分析**：提取内部相对链接作为边
- **目录层级转图**：目录结构自动生成"包含"关系边
- **配置引用分析**：分析 `include` / `depends_on` / `extends` 等字段

### Step 6：构建图数据结构

将分析结果组织为 nodes + edges 的 JSON 格式。每个节点包含：id、group（按功能分组）、label、description（含注释内容）、definitions（提取的重要定义）。每种分组有独立颜色。

### Step 7：生成交互式 HTML

使用 D3.js v7 力导向图渲染，输出自包含 HTML。

## HTML 输出规格

### 技术栈

| 类别 | 技术/库 | 作用 |
|------|---------|------|
| 核心可视化 | D3.js v7 | 力导向图布局、节点/边绘制、缩放/拖拽、动态更新 |
| 页面样式 | HTML5 + CSS3 | 侧边栏、工具提示、图例、响应式布局（暗色主题） |
| 交互功能 | 原生 JavaScript | 节点搜索、信息面板联动、缩放控制、邻居高亮 |
| 数据格式 | JSON | 内嵌的 nodes 和 edges 数据 |

### 10 项交互功能

1. **力导向布局** — 节点自动排布，同类聚集，拖拽后自动复位
2. **缩放/拖拽** — 鼠标滚轮缩放，空白区拖拽平移
3. **节点拖拽** — 单节点可拖拽，松开后固定位置
4. **邻居高亮** — 悬停节点时，关联节点和边高亮，其他淡出
5. **节点搜索** — 输入名称模糊搜索，定位居中并放大
6. **信息面板** — 点击节点，显示路径、类型、**注释内容**、定义列表
7. **图例** — 按 group 着色，显示颜色和数量统计
8. **边标签** — 悬停边时显示关系类型
9. **菜单列表** — 侧边栏列出所有节点，点击可聚焦
10. **统计信息** — 顶部显示节点数、边数、模块数

## 输出文件结构

```
project-root/
├── knowledge-graph/
│   ├── index.html       # 自包含交互式知识图谱（双击即用）
│   └── data.json        # 提取的图数据（可选，方便复查）
```

`index.html` 是**自包含**的——所有 CSS、JS（D3.js 通过 CDN 加载）和 JSON 数据都内嵌在单个 HTML 中，用户双击就能在浏览器打开。

## 常见问题

1. **图太乱**：节点数 > 100 时自动按模块聚合
2. **没有 import 关系（纯文档项目）**：按目录层级 + Markdown 内部链接构建
3. **缺少注释**：提取函数名/类名语义推断，或用路径最后两级推测功能
4. **大文件**：跳过 > 1MB 的文件
5. **CDN 失败**：使用 `https://d3js.org/d3.v7.min.js`
